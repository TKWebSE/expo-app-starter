# テスト設計・品質確認

設計書を利用者から観測できる振る舞いへ写し、TDDで実装するためのテスト方針を定義します。

## 1. TDD方針

- 要件・外部設計・内部設計に対応する失敗テストを先に書く
- 最小限の実装でテストを通し、その後に重複や構造を整理する
- 仕様変更時は設計書と対応テストを同じ変更で更新する
- 実装詳細ではなく、入力・出力・画面表示・遷移・権限を検証する
- すべてをE2Eへ寄せず、最も低い適切な層で保証する

## 2. テスト層

| 層      | 主な対象                                                     |
| ------- | ------------------------------------------------------------ |
| 単体    | Zodスキーマ、変換処理、Store、Hook、Repository               |
| 結合    | フォーム、認証状態、画面遷移、Repositoryと画面の連携         |
| DB・RLS | profilesの制約、本人だけの参照・作成・更新                   |
| E2E     | 新規登録、メール確認、ログイン、プロフィール更新、ログアウト |
| 手動    | Android・Webのレイアウト、読み上げ、キーボード、実機固有動作 |

## 3. 認証・入力検証

### パスワード

- 8文字は受理し、7文字以下は拒否する
- 64文字は受理し、65文字以上は拒否する
- 確認入力と一致しない場合は送信しない
- 空白だけの場合は拒否する
- 記号、英大文字、数字の組み合わせを強制しない
- 入力値をログ、画面、DBへ平文で残さない
- 送信中は入力とボタンを無効化し、二重送信しない

### 表示名

- 前後の空白を除いて1文字なら保存できる
- 前後の空白を除いて50文字なら保存できる
- 空文字または空白だけは拒否する
- 51文字以上は拒否する
- 日本語、英数字、記号、絵文字を保存できる
- 保存成功後は設定画面へ戻る
- 通信失敗時は入力内容を保持する

## 4. profiles・RLS

- 認証ユーザーは自分のプロフィールを参照できる
- 認証ユーザーは自分のプロフィールを作成・更新できる
- 他ユーザーのプロフィールは参照・作成・更新できない
- 未認証ユーザーはプロフィールを操作できない
- V1にはDELETEポリシーが存在しない
- 本人を含め、クライアント権限ではプロフィールを削除できない
- Repositoryにプロフィール削除APIが存在しない

## 5. 画面・操作

- 起動時の認証状態に応じてログイン画面またはホーム画面へ遷移する
- 認証失敗は画面上部のSnackbar、項目エラーは項目直下に表示する
- Snackbarは手動で閉じられ、複数通知は順番に表示される
- プロフィールに未保存変更がある場合だけ破棄確認を表示する
- ログアウト時は確認し、完了後にログイン画面へ戻る
- ProductionではUI部品確認画面へ到達できない
- Webの不明なURLは404画面を経由して適切な起点へ戻る

## 6. 環境・品質ゲート

PRとmainブランチでは、次をすべて成功させます。

- Formatter
- ESLint
- TypeScript型チェック
- 単体・結合テスト
- DB・RLSテスト
- Webビルド
- Androidビルドに必要な設定検証
- 秘密情報の混入確認

## 7. 手動確認

- Android、Web 767px以下、Web 768px以上で主要画面を確認する
- キーボードだけで操作でき、フォーカス位置が見える
- タップ領域、コントラスト、読み上げ名、エラー通知を確認する
- 文字拡大、横画面、ソフトウェアキーボード、安全領域を確認する
- Development・Staging・Productionが正しい接続先へ分離される
- コピー後にREADMEだけで別アプリとして起動できる

## 8. 完了条件

- 設計書のMVP仕様に対応するテストが存在する
- 自動品質ゲートがすべて成功する
- Android・Webの主要経路が手動確認済みである
- 他ユーザーのデータへアクセスできない
- アカウント削除・プロフィール削除が画面、Repository、RLSに含まれない
- 未決事項を残したまま期待値をテストへ固定しない

## 9. V2

- アカウント削除とプロフィール削除のUX、再認証、関連データ処理を設計する
- 削除仕様の確定後にDELETEポリシーとテストを追加する
- ライト・ダーク選択が全画面と共通UIへ即時反映され、再起動後も維持されることを確認する
- 無効・期限切れリンク、再送クールダウン、セッション期限切れからの復帰を確認する
- CI上でMigration、本人・他人・未認証・削除禁止のRLS実操作テストを実行する

### V2追加テスト

- 確認メール送信直後は再送せず、60秒後は再送できる
- メール変更とアカウント削除は現在のパスワードで再認証する
- 予期しないサインアウトではセッション期限切れを通知する
- アカウント削除FunctionはJWTの利用者と同一IDだけを管理APIへ渡す
- `auth.users`削除時に対応する`profiles`が連動削除される
- 認証済みクライアントにプロフィールDELETE権限を与えない

## V3 テスト設計

- 設定画面がプロフィールの表示名、利用規約リンク、プライバシーポリシーリンクを持つこと。
- 新規登録画面が両法務ページへのリンクを持つこと。
- メール確認必須のローカル認証設定であること。
- 登録APIがセッションを返した場合にもローカルセッションを破棄すること。
- 利用規約とプライバシーポリシーに主要条項が存在すること。
- Webビルド、型検査、Lint、ユニットテストを通過すること。
